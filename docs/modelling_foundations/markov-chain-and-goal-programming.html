13 Markov Chain and Goal Programming

A Markov chain has been incorporated into the Foundations #2 goal programming model by replacing the static habitat calculation with a dynamic component that represents forest habitat quality progression over time. Instead of treating habitat as a fixed linear function of harvested area, a transition probability matrix is used to model how forest states evolve under different levels of harvest pressure.

The steady-state distribution of this Markov process is then computed and used to derive a habitat index, which reflects the long-term ecological condition of the forest rather than just immediate post-harvest effects. This Markov-derived habitat value is embedded directly into the goal programming objective function through the habitat deviation term, allowing the optimization process to balance economic objectives (timber and revenue) with a dynamically updated measure of ecological sustainability that responds to changes in harvest intensity.

13.1 The Markov Chain Transition Probability Matrix

The Markov Chain Transition Probability Matrix is the core component of the habitat model, representing the likelihood that a forest area will transition between different habitat quality states over time. In this application, the matrix consists of three states—poor habitat, moderate habitat, and high-quality habitat—and each row contains the probabilities of moving from the current state to another state during the next time step.

Current State Poor (1) Moderate (2) High (3)
Poor (1) 0.70 0.25 0.05
Moderate (2) 0.15 0.70 0.15
High (3) 0.05 0.30 0.65

For example, a high-quality habitat area may have a strong probability of remaining in a high-quality condition, but a smaller probability of degrading to moderate or poor habitat. The probabilities in each row sum to one, ensuring that all possible future states are accounted for. As harvest pressure increases, selected transition probabilities are adjusted to reflect a greater likelihood of habitat degradation, thereby incorporating the habitat consequences of forest management activities. The resulting modified matrix is used to estimate the long-run or steady-state distribution of habitat conditions across the landscape, providing a dynamic measure of habitat sustainability that can be incorporated into the optimization process.

13.2 Markov Habitat Function

The Markov Habitat Function is used to represent how forest habitat conditions change over time in response to harvesting activity. Rather than treating habitat as a simple linear sum of habitat values, the function models the forest as occupying one of three habitat states—poor, moderate, or high quality—and uses a Markov transition matrix to describe the probability of moving between these states from one time period to the next. The base transition matrix defines the natural dynamics of the forest, while harvest pressure modifies these probabilities by reducing the likelihood that high-quality habitat remains in the highest state and increasing the probability of degradation to lower-quality states. This approach enables the model to capture the long-term ecological effects of harvesting rather than only measuring immediate habitat impacts.

To better reflect differences among management units, the function incorporates a habitat quality index based on the weighted habitat value of the harvested area. Habitat quality is combined with overall harvest pressure to calculate an ecological impact factor that adjusts the Markov transition probabilities. Harvesting large areas of habitat-rich forest therefore results in a greater reduction in future habitat quality than harvesting the same area from lower-value habitat. The adjusted transition matrix is then used to calculate the steady-state distribution of habitat states, representing the long-run proportion of the landscape expected to occur in each condition. Finally, these proportions are converted into a habitat index that serves as the habitat objective within the goal-programming optimization model, allowing the simulated annealing algorithm to balance timber production, revenue generation, and long-term habitat conservation.

forest_habitat <- function(habitat_raw, harvest_pressure, x) {
  
  P_adj <- P_base
  
  ##########################################################
  # HABITAT QUALITY INDEX
  ##########################################################
  
  if (sum(x) > 0) {
    
    # Average habitat value of harvested area
    habitat_quality <- habitat_raw / sum(x)
    
  } else {
    
    habitat_quality <- 1
    
  }
  
  # Normalize to 0–1 range
  habitat_quality <- habitat_quality / 4
  
  ##########################################################
  # COMBINED IMPACT
  ##########################################################
  
  impact <- harvest_pressure * habitat_quality
  
  ##########################################################
  # MODIFY TRANSITION PROBABILITIES
  ##########################################################
  
  P_adj[3,3] <- P_base[3,3] - 0.05 * impact
  P_adj[3,2] <- P_base[3,2] + 0.03 * impact
  P_adj[3,1] <- P_base[3,1] + 0.02 * impact
  
  ##########################################################
  # ENSURE VALID MARKOV MATRIX
  ##########################################################
  
  P_adj[P_adj < 0] <- 0
  P_adj <- P_adj / rowSums(P_adj)
  
  ##########################################################
  # STEADY-STATE DISTRIBUTION
  ##########################################################
  
  steady <- Re(eigen(t(P_adj))$vectors[,1])
  steady <- abs(steady)
  steady <- steady / sum(steady)
  
  ##########################################################
  # HABITAT INDEX
  ##########################################################
  
  habitat_markov <- sum(steady * c(1, 2, 4))
  
  habitat_markov <- habitat_markov * 100
  
  return(habitat_markov)
}

13.3 The Objective Function

This function defines the objective function for a goal programming optimization model that evaluates a proposed harvest allocation vector (x). It first calculates three performance measures—total timber harvest, revenue, and habitat—by multiplying the decision variables by their respective coefficient vectors and summing the results. It then computes the normalized deviation of each measure from its target value, expressing how far the solution is from meeting each objective relative to the target. These deviations are combined into a single weighted objective value (z) using the user-defined weights (w1, w2, and w3), allowing the relative importance of each goal to be specified.

The function also enforces operational constraints by applying large penalty values whenever the total harvested area exceeds 200 ha, or harvest in Unit 3 exceeds 80 ha. The penalties are added to the objective value, making infeasible solutions much less desirable. Finally, the function returns the total objective value, where lower values represent solutions that better satisfy the management goals while remaining within the operational constraints.


############################################################
# OBJECTIVE FUNCTION (GOAL PROGRAMMING)
############################################################

goal_function <- function(x) {
  
  harvest <- sum(timber.coef * x)
  revenue <- sum(revenue.coef * x)
  
  habitat_raw <- sum(habitat.coef * x)
  
  harvest_pressure <- sum(x) / 200
  
  habitat <- forest_habitat(
    habitat_raw = habitat_raw,
    harvest_pressure = harvest_pressure,
    x = x
  )
  
  # normalized deviations
  d.harvest <- abs(harvest - target.harvest) / target.harvest
  d.revenue <- abs(revenue - target.revenue) / target.revenue
  d.habitat <- abs(habitat - target.habitat) / target.habitat
  
  z <- (w1 * d.harvest) +
    (w2 * d.revenue) +
    (w3 * d.habitat)
  
  ##########################################################
  # PENALTIES 
  ##########################################################
  
  penalty <- 0
  
  # non-negativity constraint (IMPORTANT FIX)
  if (any(x < 0)) {
    penalty <- penalty + 1000 * sum(abs(x[x < 0]))
  }
  
  # area constraint
  if (sum(x) > 200) {
    penalty <- penalty + 1000 * (sum(x) - 200)
  }
  
  # unit 3 constraint
  if (x[3] > 80) {
    penalty <- penalty + 1000 * (x[3] - 80)
  }
  
  return(z + penalty)
}

13.4 The SANN Optimization and RESULTS

This code uses simulated annealing to search for a harvest allocation that minimizes the goal_function while balancing multiple management objectives and discouraging constraint violations. The optim() function is called with the "SANN" method, starting from an initial solution of 50 ha in each management unit (par = c(50, 50, 50)) and running for up to 50,000 iterations with an initial temperature of 50.

13.5 Results

The integrated goal programming and Markov chain model successfully achieves full attainment of timber and revenue targets, indicating that the economic objectives are achievable under the current harvesting regime. However, the Markov-based habitat component introduces a dynamic quality constraint that reduces long-term habitat availability under sustained harvest pressure. As a result, habitat targets are not fully satisfied, highlighting a trade-off between economic optimisation and the maintenance of habitat quality.

The solution represents a Pareto-efficient compromise where economic goals are fully met at the expense of moderate habitat quality deviation.A Pareto-efficient compromise is a solution to a multi-objective problem where you cannot improve one objective (like harvest, or revenue) without making at least one other objective (like habitat or sustainability) worse.

In other words, it represents a “best possible trade-off” between competing goals: the solution is efficient because no objective can be improved independently, but it is still a compromise because not all goals are fully satisfied at their ideal targets. In the model, for example, a Pareto-efficient outcome might fully meet harvest and revenue targets while only partially meeting habitat goals, or improve habitat slightly only by reducing economic returns.
