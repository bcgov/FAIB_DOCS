6 Cellular Automata and the Game of Life

6.1 Introduction

Cellular automata are mathematical and computational models that simulate how complex systems evolve over time through the interaction of simple components. They consist of a grid of cells, where each cell exists in one of a finite number of states, such as alive or dead. At each time step, every cell updates its state simultaneously according to a predefined set of rules based on the states of its neighbouring cells.

Despite their simple local rules, cellular automata can generate remarkably complex and unpredictable behaviour. Depending on the initial configuration and the update rules, they may produce stable structures, repeating patterns, moving formations, or highly chaotic dynamics. This ability to create intricate behaviour from simple interactions has made cellular automata an important tool for studying emergence, self-organization, and complexity in natural and artificial systems.

Cellular automata have applications across many scientific disciplines, including physics, biology, computer science, and mathematics. They have been used to model phenomena such as population growth, forest fires, disease transmission, traffic flow, crystal formation, and fluid dynamics. One of the most well-known examples is Conway’s Game of Life, which demonstrates how a few straightforward rules can give rise to rich and unexpected patterns, illustrating the power of local interactions to produce complex global behaviour.

6.2 Wolfram

Stephen Wolfram classified the long-term behaviour of cellular automata into four broad classes based on the types of patterns they produce from different initial conditions. This classification provides a useful framework for understanding how simple rules can generate behaviours ranging from complete stability to apparent randomness. Although originally developed for one-dimensional cellular automata, the classification has also been applied more broadly to other cellular automaton systems.

Class I cellular automata quickly evolve to a uniform, stable state in which all cells become identical. Class II automata develop simple, repeating, or stable structures that persist over time, often resulting in oscillating or fixed patterns. Class III automata produce chaotic, pseudo-random behaviour characterized by irregular patterns and strong sensitivity to initial conditions. Although fully deterministic, their evolution often appears random, making certain Class III rules useful for pseudo-random number generation and the study of complex dynamical systems.

Class IV cellular automata exhibit behaviour that lies between order and chaos. They generate persistent, localized structures that move and interact in complex ways while maintaining an overall balance between stability and unpredictability. This class is considered the most computationally interesting because it is capable of supporting information transfer and complex interactions. Conway’s Game of Life is one of the best-known examples of a Class IV cellular automaton, producing familiar structures such as still lifes, oscillators, and moving “gliders.” The rich dynamics of Class IV systems have made them an important area of research in complexity science, artificial life, and theoretical computer science.

6.2.1 Codes Notes

6.2.1.1 Parameters

These parameters define the size of the Game of Life grid (50 × 50 cells), the number of generations to simulate (500), and the initial probability (20%) that any given cell is alive when the simulation begins.

6.2.1.2 Convert rule number to lookup table

The make_rule() function converts a Wolfram elementary cellular automaton rule number (0–255) into its 8-bit binary representation and associates each bit with one of the eight possible three-cell neighbourhood patterns (111 through 000). This creates a lookup table that determines the next state of a cell for every possible neighbourhood #configuration. The resulting rule table is then used to evolve the cellular automaton from one generation to the next.

Elementary cellular automata have 256 possible rules because there are eight possible three-cell neighbourhood configurations, and each configuration can produce either a 0 or a 1 in the next generation. This gives 256 unique combinations, which are numbered from Rule 0 to Rule 255.

6.2.1.3 Compute next generation

The update_generation() function applies a Wolfram cellular automaton rule to every cell in the current generation by examining the three-cell neighbourhood consisting of the left, centre, and right cells. For each neighbourhood pattern, it looks up the corresponding output in the rule table and constructs a new array representing the next generation of the automaton.

6.2.1.4 Run simulation

The simulate_ca() function runs a complete Wolfram elementary cellular automaton simulation by first converting the selected rule number into a rule table and then initializing a one-dimensional grid with a single live cell at its centre. It repeatedly applies the automaton rule to generate each successive generation and records the state of every cell after each update. The results are stored in a matrix called history, where each row represents a generation and each column represents a cell position, allowing the evolution of the automaton to be visualized over time.

make_rule <- function(rule_number) {
  
  bits <- rev(as.integer(intToBits(rule_number)[1:8]))
  
  names(bits) <- c(
    "111", "110", "101", "100",
    "011", "010", "001", "000"
  )
  
  bits
}

###############################################################################################################
# Compute next generation
##################################################################################################################

update_generation <- function(cells, rule) {
  
  n <- length(cells)
  next_cells <- integer(n)
  
  for(i in seq_len(n)) {
    
    left   <- cells[((i - 2) %% n) + 1]
    center <- cells[i]
    right  <- cells[(i %% n) + 1]
    
    pattern <- paste0(left, center, right)
    
    next_cells[i] <- rule[pattern]
  }
  
  next_cells
}

simulate_ca <- function(
    rule_number = 30,
    width = 101,
    steps = 100
) {
  
  rule <- make_rule(rule_number)
  
  history <- matrix(
    0,
    nrow = steps,
    ncol = width
  )
  
  current <- rep(0, width)
  
  # single seed
  current[(width + 1) %/% 2] <- 1
  
  history[1, ] <- current
  
  for(t in 2:steps) {
    
    current <- update_generation(
      current,
      rule
    )
    
    history[t, ] <- current
  }
  
  history
}

6.3 The Game of Life

Conway’s Game of Life is a two-dimensional cellular automaton devised by mathematician John Conway in 1970. The system consists of a grid of cells, each of which can be either alive or dead. At each time step, the state of every cell is updated according to a simple set of rules based on the number of living neighbours: a live cell survives with two or three neighbours, a dead cell becomes alive with exactly three neighbours, and all other cells either die or remain dead. Despite the simplicity of these rules, the Game of Life produces a remarkable range of behaviours, from stable structures and repeating oscillators to complex moving patterns known as gliders.

The Game of Life is called the “Game of Life” because it was designed to simulate, in a highly simplified way, some of the fundamental processes associated with living organisms and populations. Cells on a grid can be either alive or dead, and simple rules governing birth, survival, and death determine how the pattern evolves over time. Conway chose the name because these interactions resemble biological concepts such as reproduction, overcrowding, and extinction, yet the system requires no further input once the initial configuration is created. The remarkable emergence of complex, lifelike behaviour from such simple rules illustrates how intricate patterns and organization can arise naturally from local interactions, making the name both descriptive and symbolic of the system’s biological inspiration.

The Game of Life is often cited as one of the most important examples of emergence, where complex behaviour arises from simple local interactions. Common patterns include still lifes that remain unchanged indefinitely, oscillators that repeat on a fixed cycle, and spaceships that propagate across the grid. Interactions among these structures can generate increasingly intricate configurations, allowing large-scale organization to emerge without any centralized control. This property has made the Game of Life a popular model for studying self-organization, complex systems, and artificial life.

Within Stephen Wolfram’s classification of cellular automata, the Game of Life is generally regarded as a Class IV system. The Game of Life exemplifies Class IV behaviour because it supports persistent structures that move, collide, and transform while maintaining coherent identities over extended periods. These interactions can be used to construct logical gates, memory elements, and computational circuits, leading to the landmark result that the Game of Life is Turing complete. (Turing completeness is the property of a system that enables it to perform any computation that a general-purpose computer can carry out, provided sufficient time and memory are available. The Game of Life meets this criterion because patterns of living cells can be arranged to create logical gates, memory elements, and information-carrying structures, allowing the automaton to simulate the operation of a universal computer despite being governed by only a few simple rules)

In Wolfram’s rule-based framework, the one-dimensional cellular automaton Rule 110 serves as the canonical Class IV example, while the Game of Life can be viewed as its two-dimensional counterpart. Together, they demonstrate how relatively simple rules can generate computationally universal systems that lie at the boundary between order and chaos.

6.3.1 Code Notes

6.3.1.1 Count Neighbors

The count_neighbors() function calculates the number of living neighbours surrounding each cell in the Game of Life grid by creating shifted versions of the grid in all eight directions (up, down, left, right, and the four diagonals). It uses periodic (wrap-around) boundary conditions so that cells on one edge interact with cells on the opposite edge, then sums the eight shifted grids to produce a #matrix containing the neighbour count for every cell.

6.3.1.2 Next Generation

The next_generation() function applies Conway’s Game of Life rules to determine which cells survive, die, or are born in the next generation based on the number of living neighbours calculated by count_neighbors(). It then creates and returns a new grid in which live cells with two or three neighbours survive, dead cells with exactly three neighbours become alive, and all other cells are dead.

6.3.1.3 Initialize Grid

This code initializes the Game of Life grid by creating a grid_size × grid_size matrix in which each cell is randomly assigned as alive (1) with probability equal to density or dead (0) otherwise.

6.3.1.4 Run Simulations

This code runs the Game of Life simulation for the specified number of generations, recording the coordinates of all living cells at each time step and storing them in a list of data frames. After recording the current state, the grid is updated to the next generation using Conway’s rules. Finally, all recorded generations are combined into a single data frame (life_df), which contains the positions of living cells and the generation in which they occur, making the data suitable for visualization and animation.

grid_size <- 50
generations <- 100
density <- 0.20

################################# Functions##################################################################
#COUNT Neighbours
###############################################################################################################
count_neighbors <- function(g) {

  up    <- rbind(g[nrow(g), ], g[-nrow(g), ])
  down  <- rbind(g[-1, ], g[1, ])

  left  <- cbind(g[, ncol(g)], g[, -ncol(g)])
  right <- cbind(g[, -1], g[, 1])

  up_left    <- cbind(up[, ncol(g)], up[, -ncol(g)])
  up_right   <- cbind(up[, -1], up[, 1])

  down_left  <- cbind(down[, ncol(g)], down[, -ncol(g)])
  down_right <- cbind(down[, -1], down[, 1])

  up + down +
    left + right +
    up_left + up_right +
    down_left + down_right
}
###########################################################################################################
#NEXT GENERATION
##############################################################################################################3
next_generation <- function(g) {

  nbrs <- count_neighbors(g)

  survive <- (g == 1L) &
    (nbrs %in% c(2L, 3L))

  born <- (g == 0L) &
    (nbrs == 3L)

  matrix(
    as.integer(survive | born),
    nrow = nrow(g),
    ncol = ncol(g)
  )
}

The following population curve shows an initial rapid decline in the number of living cells as unstable configurations are eliminated. Following this transient phase, the system exhibits sustained fluctuations in population size, indicating ongoing cycles of cell birth and death. Rather than converging to a homogeneous or periodic state, the automaton maintains a dynamic equilibrium characterized by interacting structures and persistent activity. This behaviour is consistent with the complex dynamics typically associated with Wolfram’s Class IV cellular automata.
