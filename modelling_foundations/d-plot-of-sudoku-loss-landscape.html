5 3D plot of Sudoku loss landscape

This script snippet analyzes and visualizes how the Sudoku solver explores the solution space over time.

After the optimization runs are complete, the script investigates the structure of the search process by applying Principal Component Analysis (PCA) to the recorded Sudoku states.Before performing PCA, any cells whose values never changed are removed since they provide no information about the search dynamics. PCA projects the remaining high-dimensional Sudoku configurations into a three-dimensional space, allowing the optimization trajectories to be visualized. The script then identifies potential lock-in points, where the solver begins making very little progress, by detecting when the average rate of error reduction falls below a specified threshold over a sliding window of iterations.

These points are highlighted on the visualizations. Three complementary visual outputs are produced: a two-dimensional PCA plot showing the optimization landscape colored by error with each run’s trajectory overlaid and lock-in points marked; an animated version illustrating how each optimization path evolves over time; and an interactive three-dimensional PCA plot that enables exploration of the trajectories from different viewing angles. Together, these visualizations provide insight into how simulated annealing navigates the Sudoku solution landscape, whether different runs converge toward similar regions, and where the search begins to stagnate.

5.1 Why use PCA

Principal Component Analysis (PCA)is a statistical technique that simplifies complex datasets by finding the main patterns of variation within the data. Imagine you have data with many variables—in this case, each Sudoku board (Each candidate solution is a different Sudoku board configuration) is represented by 81 cell values,meaning every board exists as a point in an 81-dimensional space. Humans cannot visualize spaces with that many dimensions, so PCA transforms the data into a new set of variables, called principal components, that capture as much of the important information as possible while using far fewer dimensions.

PCA works like finding the best viewing angle for a complicated object. Imagine looking at a tangled ball of string. From one direction it might look like a messy blob,but if you rotate it to the right angle, you can clearly see its overall shape. PCA automatically finds these “best angles” by identifying the directions in which the data varies the most. The first principal component (PC1) captures the greatest amount of variation, the second principal component (PC2) captures the next greatest amount (while being independent of PC1), and the third principal component (PC3) captures the next largest remaining variation.

For the Sudoku optimization algorithm, each recorded Sudoku board represents one point in a very high-dimensional search space. PCA projects these points into two or three dimensions while preserving as much of their relative structure as possible. This allows the optimization process to be visualized as a path through a landscape. States that appear close together inthe PCA plot are generally similar Sudoku configurations, while distant points represent substantially different boards. As the simulated annealing algorithm progresses, its trajectory through the PCA space illustrates how it explores different regions of the search space before converging toward a solution.

It is important to remember that PCA does not change or improve the data—it only provides a simpler representation of it. Because the projection reduces many dimensions to just two or three, some information is inevitably lost. However, if the first few principal components explain most of the variation in the data, the resulting visualization still provides an accurate picture of the overall search behavior. This makes PCA a valuable tool for understanding optimization algorithms, identifying clusters of similar solutions, observing convergence, and detecting when the search becomes trapped in a local optimum.

The 3d plot takes a bit of time to run ~ 5 minutes

library(plotly)
library(gganimate)

# ---------------------------
# CORE FUNCTIONS:
# fixSudokuValues, getBlocks, fillBlocks,
# swapCells, calcErrors
# ---------------------------

# ---------------------------
# 1. Collect samples (single run)
# ---------------------------
collectSamples <- function(startingSudoku, steps = 1000) {
  fixed <- fixSudokuValues(startingSudoku)
  blocks <- getBlocks()
  
  s <- fillBlocks(startingSudoku, blocks)
  
  sigma <- 2
  cooling <- 0.995
  
  states <- matrix(NA, nrow = steps, ncol = 81)
  errors <- numeric(steps)
  
  for (i in 1:steps) {
    block <- blocks[[sample(1:9, 1)]]
    new <- swapCells(s, fixed, block)
    
    d <- calcErrors(new) - calcErrors(s)
    
    if (d <= 0 || runif(1) < exp(-d / sigma)) {
      s <- new
    }
    
    sigma <- sigma * cooling
    
    states[i, ] <- as.vector(s)
    errors[i] <- calcErrors(s)
  }
  
  list(states = states, errors = errors)
}

# ---------------------------
# 2. Multiple runs
# ---------------------------
n_runs <- 3
all_states <- NULL
all_errors <- NULL
run_id <- NULL

for (r in 1:n_runs) {
  res <- collectSamples(startingSudoku, 1000)
  
  all_states <- rbind(all_states, res$states)
  all_errors <- c(all_errors, res$errors)
  run_id <- c(run_id, rep(r, length(res$errors)))
}

# ---------------------------
# 3. Remove constant columns
# ---------------------------
vars <- apply(all_states, 2, var)
states_filtered <- all_states[, vars > 1e-6]

# ---------------------------
# 4. PCA (3D)

pca <- prcomp(states_filtered, scale. = TRUE)

proj <- data.frame(
  PC1 = pca$x[,1],
  PC2 = pca$x[,2],
  PC3 = pca$x[,3],
  error = all_errors,
  step = rep(1:(nrow(all_states)/n_runs), n_runs),
  run = factor(run_id)
)

# ---------------------------
# 5. LOCK-IN DETECTION
# ---------------------------
detect_lockin <- function(errors, window = 30, threshold = 0.01) {
  for (i in 1:(length(errors) - window)) {
    slope <- abs(errors[i + window] - errors[i]) / window
    if (slope < threshold) return(i)
  }
  return(NA)
}

proj$lockin <- NA

for (r in levels(proj$run)) {
  idx <- proj$run == r
  lock_point <- detect_lockin(proj$error[idx])
  
  if (!is.na(lock_point)) {
    proj$lockin[which(idx)[lock_point]] <- TRUE
  }
}

# ---------------------------
# 6. 2D Plot (multiple runs + lock-in)
# ---------------------------
p2d <- ggplot(proj, aes(PC1, PC2)) +
  
  # Landscape: continuous error coloring
  geom_point(aes(color = error), alpha = 0.4, size = 1.2) +
  
  # Paths: no color mapping (use grouping only)
  geom_path(aes(group = run), color = "red", linewidth = 1, alpha = 0.7) +
  
  # Lock-in markers
  geom_point(data = subset(proj, lockin == TRUE),
             color = "black", size = 3, shape = 8) +
  
  scale_color_viridis_c() +
  
  labs(
    title = "Sudoku Loss Landscape (PCA)",
    subtitle = "Multiple runs + lock-in detection",
    color = "Error"
  ) +
  
  theme_minimal()

print(p2d)


# ---------------------------
# 7. Animated PCA trajectory
# ---------------------------
p_anim <- ggplot(proj, aes(PC1, PC2, group = run, color = run)) +
  geom_path(alpha = 0.7) +
  geom_point(size = 5) +
  labs(title = "Annealing Trajectory (Step: {frame_time})") +
  transition_time(step) +
  theme_minimal()

animate(p_anim, fps = 10)


# ---------------------------
# 8. 3D PCA Plot (interactive)
# ---------------------------

p3d <- plot_ly(
  proj,
  x = ~PC1,
  y = ~PC2,
  z = ~PC3,
  color = ~error,
  colors = "viridis",   # ✅ FIX: lowercase
  type = "scatter3d",
  mode = "lines+markers",
  split = ~run
)

5.2 Interpretation

The trajectories from different runs converge toward similar dense clusters, indicating that the solver consistently finds low-error basins in the search landscape, even though multiple paths are taken. The presence of branching structures and distinct clusters suggests a rugged landscape with several local minima, while the increasingly tight grouping of low-error points reflects the annealing process “locking in” as temperature decreases. PCA plays a crucial role here by revealing these global patterns—such as convergence, clustering, and basin formation—that would be invisible in the raw high-dimensional representation.

Overall, the plot demonstrates that simulated annealing effectively reduces error and guides the solver toward stable regions of the solution space, while also highlighting the inherent complexity and multi-modal structure of the Sudoku optimization problem.
